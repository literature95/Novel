# Snowflake Novel Architecture v2.0

**规范数据模型**（JSON Schema draft-07）：面向长篇叙事与协同创作的雪花嵌套结构。  
**运行时扁平格式**见 [`小说结构模型v2.1.md`](./小说结构模型v2.1.md)，由 `src/lib/project.js` 读写；导入导出 JSON 使用 v2.1 形状。

与 [`小说流程.md`](./小说流程.md)、[`状态节点.md`](./状态节点.md)、[`README.local.md`](./README.local.md) 一致。

---

## 顶层模块（v2.0）

| 模块 | 说明 |
|------|------|
| `story_arcs` | 三幕宏观节奏 |
| `plot_lines` | 五条并行叙事线 |
| `volumes` | 分册与章节树 |
| `knowledge_graph` | 角色、地点、力量、时间线 |
| `foreshadow_graph` | 伏笔节点、边、状态机 |
| `collaboration_board` | 协同分工（单人可跳过） |
| `constraints_engine` | 规则与违规日志 |
| `progress_aggregator` | 由场景汇总的进度 |
| `integrity_checkpoints` | 完整性检查开关 |

---

## v2.0 ↔ 运行时 (v2.1) 字段映射

| v2.0 路径 | 运行时字段 (`project.js`) | 节点 |
|-----------|---------------------------|------|
| `knowledge_graph.characters[]` | `characters[]` | N-1.1 |
| `knowledge_graph.locations[]` | `locations[]` | N-1.2 |
| `knowledge_graph.magic_system[]` | `magicSystem[]` | N-1.3 |
| `knowledge_graph.timeline` | `timeline[]` | N-1.4 |
| `constraints_engine.rules[]` | `constraints[]` | N-1.5 |
| `story_arcs.act_*` | `acts[]` | N-2.1 |
| `plot_lines.*` | `plotLines[]` | N-2.2 |
| `foreshadow_graph.nodes` | `foreshadows[]` | N-2.3 / N-6.1 |
| `volumes[]` | `volumes[]` | N-4.1 |
| `volumes[].chapters[]` + `chapter_metadata` | `chapters[]` | N-4.2 |
| `scenes[]`（按章嵌套） | `scenes: { [chapterId]: Scene[] }` | N-5.1 |
| `collaboration_board` | `collaboration`（writers / chapterAssignments / handoverNotes / syncLog） | N-3.1 / N-6.3 |
| `progress_aggregator` | `lib/progress-aggregator.js` 计算 | N-5.1 / N-7.4 |
| `integrity_checkpoints` | —（规划） | N-6.2 |
| — | `phases[]` `staleMarkers` `meta` `settings` | 工作流 / UI |

约束类型：`absolute` / `soft` / `guideline` ↔ 运行时 `constraints[].type`。  
伏笔状态：v2.0 状态机含 `drafted`→`planted`→…→`resolved`；运行时见 `foreshadow-engine.js`。
