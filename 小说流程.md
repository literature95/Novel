| 七 | 修订闭环与交付 | N-7.1～N-7.4 | 全模型终审 | `FinalReport` 聚合 | 伏笔/弧线/规则兑现，N-7.4 综合健康度决策 |

## 与前端视图的对应

| 视图 | 侧边栏入口 | 主要覆盖阶段 |
|------|------------|--------------|
| 项目首页 | 展示区 · 首页 | 总览、导入导出、进度摘要 |
| 小说结构生成 | 工作区 · 结构 | 阶段一～四、六、七（节点卡片 + 面板） |
| 小说内容创作 | 工作区 · 内容 | 阶段五 N-5.1（场景编辑 + 校验） |
| AI 模型设置 | 系统 · 设置 | `project.settings`（API Key / 模型） |

**结构页主路径**：节点卡片 **「✦ 完善模型」** → `NodeWorkflowPanel`（`node-runner` 六步：加载上下文 → AI 生成 → 校验 → 预览确认 → 锁定写入 `apply-node-data`）。

## 六步节点循环（所有 N-x.x 共用）

详见 [`状态节点.md`](./状态节点.md) § 每个节点的内部标准流程。

1. **加载上下文** — `lib/context-orchestrator.js`，单次 ≤ 8KB  
2. **AI 生成** — `lib/ai-client.js`（无 Key 时 `node-runner` 回退模拟数据）  
3. **自动校验** — `schema-validator`（结构节点）；场景另走 `constraints-engine`  
4. **Web 展示** — `NodeWorkflowPanel` JSON 预览 + 各 `*Panel` 手编  
5. **人工确认** — 预览区可改 JSON 后再校验/写入  
6. **锁定写入** — `apply-node-data` → `store` `APPLY_NODE_GENERATION` + `completeNode`；持久化 `localStorage`
